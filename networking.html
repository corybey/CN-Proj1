1) DNS (Domain Name System)

DNS translates a human-readable domain name into an IP address. When a user visits the website URL, DNS resolves the domain to the hosting server’s address so the browser can send traffic to the correct destination. 

nslookup YOUR_GITHUB_PAGES_DOMAIN

Example: nslookup corybey.github.io

2) IP Addressing (IPv4 / IPv6)

IP addressing allows devices and servers to identify one another across the Internet. Once DNS completes resolution, traffic is routed using IP addresses. 

nslookup YOUR_GITHUB_PAGES_DOMAIN
ping YOUR_GITHUB_PAGES_DOMAIN

If IPv6 appears in your results, mention it in your report.

3) HTTPS Security

This website uses HTTPS to encrypt traffic between the client and server using TLS. HTTPS protects confidentiality and integrity by helping prevent interception and tampering during transmission.

  • Look for the lock icon in the browser
  • Use DevTools → Network to verify requests use https://
  • Capture a screenshot for your report

4) XSS Prevention Through Sanitized Rendering

The interactive feedback board accepts user input, which creates a possible XSS attack surface. To reduce this risk, all submitted content is sanitized before it is rendered back onto the page.

  • User input is trimmed and validated
  • DOMPurify sanitizes data before rendering
  • Unsafe HTML or script content is not executed in the browser
Example: If a user submits a script tag, the page will display safe content instead of executing malicious JavaScript.

5) Firestore Database Integration

Firestore makes the website dynamic by allowing visitor responses to be stored and retrieved in real time. Instead of showing only static HTML content, the site now loads recent feedback entries from a cloud database.

  • Collection used: feedback
  • Fields stored: name, topic, message, createdAt
  • Data is loaded dynamically onto the home page

6) Performance Optimization

Several front-end optimizations were added to improve load behavior and responsiveness.

  • JavaScript is loaded with module/deferred behavior
  • DOMPurify is loaded with defer
  • Meta description added for better page structure
  • Only essential assets are loaded
  • Reusable styling keeps the site lightweight

7) Challenges & Solutions

  • Challenge: Making a static GitHub Pages site more dynamic
  • Solution: Connected the front end directly to Firestore
  • Challenge: Accepting user input securely
  • Solution: Sanitized rendered content using DOMPurify
  • Challenge: Keeping the site lightweight and fast
  • Solution: Used minimal dependencies and deferred script loading
