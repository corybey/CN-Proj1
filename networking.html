1) DNS (Domain Name System)

DNS translates a human-readable domain name into an IP address. When a user visits the website URL, DNS resolves the domain to the hosting server’s IP.

Evidence (paste screenshot in report):

nslookup YOUR_GITHUB_PAGES_DOMAIN

Example: nslookup yourusername.github.io

2) IP Addressing (IPv4 / IPv6)

A domain may resolve to IPv4 addresses, IPv6 addresses, or both. This shows how the Internet routes traffic to the correct server using IP addressing.

Evidence commands:

nslookup YOUR_GITHUB_PAGES_DOMAIN
ping YOUR_GITHUB_PAGES_DOMAIN

If IPv6 appears, mention it. If only IPv4 appears, say so.

3) Protocols: HTTP vs HTTPS

HTTP sends data in plaintext. HTTPS uses TLS to encrypt the connection, protecting confidentiality and integrity. Modern browsers prefer HTTPS for public websites.

Evidence: Use browser DevTools → Network tab → click the request.

  • Status code (ex: 200 OK)
  • Request/Response headers
  • Protocol: HTTPS

4) Security Feature Implemented

This website uses HTTPS through GitHub Pages. HTTPS uses TLS to encrypt traffic between the client and server, helping prevent eavesdropping and tampering.
